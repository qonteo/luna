Persons API¶

class luna_api.app.rest_handlers.create_person_handler.PersonCreateHandler(application, request, **kwargs)[source]¶

Handler for person creation. To work with this handler you should authorize in the account and it must be active.

get()[source]¶

Resource to get all persons.

POST /storage/persons?page=1&page_size=10¶
Optparam page:

A number of page. Minimum 1, default 1.
Optparam page_size:
 

Persons count on page. Minimum 1, maximum 100, default 10.
Request Headers:
 
  • LUNA-Request-Id – request id

Example request:

Request Headers:
 
  • Authorization – basic authorization

or

Request Headers:
 
  • X-Auth-Token – ‘16fd2706-8baf-433b-82eb-8c7fada847da’
Query Parameters:
 
  • user_data – user data or part of user data to search by it

Example response:

HTTP/1.1 200 Ok
Vary: Accept
Content-Type: application/json
LUNA-Request-Id: 1516179740,c06887a2
Status Codes:
  • 200 OK – person ids and number of persons are received successfully

Request for new account creation

POST /storage/persons¶
Request Headers:
 
  • LUNA-Request-Id – request id

Example request:

Request Headers:
 
  • Authorization – basic authorization

or

Request Headers:
 
  • X-Auth-Token – ‘16fd2706-8baf-433b-82eb-8c7fada847da’
 
POST /storage/persons HTTP/1.1
Accept: application/json

You can attach user data in format user_data and/or external id in format external_id.

Example response:

HTTP/1.1 201 Created
Vary: Accept
Content-Type: application/json
LUNA-Request-Id: 1516179740,c06887a2
id_person¶
Object Properties:
 
  • person_id (uuid4) – id of created person
JSON Example
{
    "person_id": "8a5d4370-0994-48a1-9a40-79c3ca59d743"

Error message is returned on format server_error.

Status Codes:
  • 400 Bad Request – field user_data is too large
  • 400 Bad Request – field user_data has wrong type, string type is required
  • 400 Bad Request – field external_id is too large
  • 400 Bad Request – field external_id has wrong type, string type is required
  • 500 Internal Server Error – internal server error
