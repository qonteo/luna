Registration¶

class luna_api.app.rest_handlers.registration_handler.RegistrationHandler(application, request, **kwargs)[source]¶

Handler to register accounts

post()[source]¶

Resource is available at ‘/accounts’

POST /accounts¶

Request for registration.

Example request:

Request Headers:
 
  • LUNA-Request-Id – request id
 
POST /accounts HTTP/1.1
Accept: application/json
json for registration¶
Object Properties:
 
  • organization_name (user_name) – name of registration organization
  • email (email) – email of registration organization
  • password (string) – password for access to account (minlength=8, maxlength=32)
JSON Example
{
    "organization_name": "john37",
    "email": "sean11@turner.com",
    "password": "DgFIXkSMclPFZiJcupgm"
}

Example response:

HTTP/1.1 201 Created
Vary: Accept
Content-Type: application/json
LUNA-Request-Id: 1516179740,c06887a2
response¶
Object Properties:
 
  • token (uuid4) – token for access
JSON Example
{
    "token": "9dcd5870-411e-4a35-8c63-d50c8afc2738"
}

Error message is returned in format server_error.

Status Codes:
  • 201 Created – registration is successful
  • 409 Conflict – e-mail is already taken
  • 400 Bad Request – request does not contain json
  • 400 Bad Request – password is too short or too long
  • 400 Bad Request – password field was not found in json
  • 400 Bad Request – password is empty
  • 400 Bad Request – email could not be parsed
  • 400 Bad Request – email was not found in json
  • 400 Bad Request – email is empty
  • 400 Bad Request – organization_name could not be parsed
  • 400 Bad Request – organization name is empty
  • 500 Internal Server Error – internal server error
