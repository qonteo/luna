Person API¶

class luna_api.app.rest_handlers.person_handler.PersonHandler(application, request, **kwargs)[source]¶

Handler to operate with existing person. First you have to authorize and account must be active.

delete(person_id)[source]¶

Resource for person deletion.

Person and all its lists will be blocked for deletion/objects addition during person deletion. After deletion, all objects must be unblocked (regardless, deletion was successful or not).

PATCH /storage/persons/{person_id}¶
Parameters:
  • person_id – person id

Example request:

Request Headers:
 
  • Authorization – basic authorization

or

Request Headers:
 
  • X-Auth-Token – ‘16fd2706-8baf-433b-82eb-8c7fada847da’

Example response:

HTTP/1.1 204 Ok
Vary: Accept
LUNA-Request-Id: 1516179740,c06887a2
Status Codes:
  • 204 No Content – person is deleted successfully

Error message is returned in format server_error.

Status Codes:
  • 404 Not Found – person not found
  • 403 Forbidden – person is currently blocked
  • 500 Internal Server Error – internal server error
