Installation¶

Packages and requirements installation¶

For the Facestreammanager2 operation, one needs to install packages and requirements:

  1. Install python 3.5 or higher;
  2. Install Elasticsearch 6 (read here);
  3. Install latex
sudo yum -y install texlive texlive-*.noarch
  1. Install Python libraries:

Note: Virtual environment is strongly recommended for installation.

  • Create a virtual environment
python3.x -m venv venv
  • Activate the virtual environment by the command
source ./venv/bin/activate

for deactivation enter

deactivate
  • Go to the folder with the project
cd facestreammanager2
  • Install dependencies by the command
pip install -r requirements.txt

Setup and Startup¶

To start the Facestreammanager2, one should:

  1. Start Elasticsearch by the command ./elasticsearch-6.x.x/bin/elasticsearch;
  2. Check and change the service settings in the configuration file ./configs/config.conf (if necessary):
ELASTICSEARCH_HOST="http://localhost"         #: Elasticsearch address
ELASTICSEARCH_PORT=9200                       #: Elasticsearch port


LUNA_API_HOST = "http://127.0.0.1"            #: LUNA API address
LUNA_API_PORT = 5000                          #: LUNA API port
LUNA_API_API_VERSION = 4                      #: LUNA API api version
LUNA_API_TOKEN = ""                           #: LUNA API account token


LUNA_CORE_HOST = "http://127.0.0.1"           #: LUNA CORE address
LUNA_CORE_PORT = 8083                         #: LUNA CORE port
LUNA_CORE_API_VERSION = 14                    #: LUNA CORE api version

LUNA_IMAGE_STORE_HOST = "http://127.0.0.1"              #: LUNA IMAGE STORE address
LUNA_IMAGE_STORE_PORT = 5020                            #: LUNA IMAGE STORE port
LUNA_IMAGE_STORE_API_VERSION = 1                        #: LUNA IMAGE STORE api version
LUNA_IMAGE_STORE_WARPS_BUCKET = "luna-core-warp-images" #: LUNA IMAGE STORE bucket for storing warped images from extractor


LOG_LEVEL = "DEBUG"                           #: level of debug print, by priority: ERROR, WARNING, INFO, DEBUG

FOLDER_WITH_LOGS="./"                         #: folder, where logs are saved

USE_LATEX = 0                                 #: whether to use latex

SAVE_INPUT_IMAGES = 0                         #: whether to save input event images
  1. Create Elasticsearch database by the python make_index.py script;
  2. Start the Facestreammanager2 by the command python run.py. The Facestreammanager2 provides application and analytics modules, that start at ports 5100 and 5101 respectively. You can assign different ports using parameters events_port and analytics_port if needed, also you can change default config file location through config parameter, for example python run.py –config=./configs/new_config.conf –events_port=8080 –analytics_port=8081.

Tests¶

  1. Ensure that both modules and Luna API are running.
  2. Check and, if necessary, change the test settings in the tests/system_tests/config.py configuration file:
  • ELASTICSEARCH_URL = “<Elasticsearch url>”
  • LUNA_API_HOST = “<Luna API host>”
  • LUNA_API_PORT = <Luna API port>
  • LUNA_API_API = <Luna API api version>
  • LUNA_AUTH_TOKEN = “<Luna API token>”
  • FSM2_URL = “<Facestreammanager2 url>”
  • FSM2_API_VERSION = <Facestreammanager2 api version>
  1. Run tests by the command
python -m unittest tests.system_tests.unittests_main

For api reference usage see ./raml/fsm2.html
