Database structure¶

Elasticsearch is a search engine based on Lucene. It provides a distributed, multitenant-capable full-text search engine with an HTTP web interface and schema-free JSON documents.

The entire database is divided into indexes, each of which has one mapping.

Six indices with one mapping “/doc” in each are used in the current version of the project. Indexes are listed below:

/handlers¶

/handlers structure is used to store ‘handlers’ objects - event processing policies, read RAML for more information.

You can see index structure and example here /handlers objects’ structure.

/events¶

/events structure is used to store ‘events’ objects - <description>.

You can see structure and example here /events objects’ structure.

/groups¶

/groups structure is used to store ‘groups’ objects - <description>.

You can see structure and example here /groups objects’ structure.

/tasks_counter¶

/tasks_counter structure is used to store the only ‘tasks_counter’ object (“/tasks_counter/doc/0”) - incremental counter of the last created task id.

You can see structure and example here /tasks_counter object structure.

/tasks¶

/tasks structure is used to store ‘tasks’ objects - abstract long-time instructions, that Facestreammanager2 receive and perform.

You can see structure and example here /tasks objects’ structure.

/tasks_done¶

/tasks_done structure is used to store ‘tasks_done’ objects - ended tasks (succeed, failed or canceled) with corresponding result.

You can see structure and example here /tasks_done objects’ structure.

Database objects’ structure¶

/handlers objects’ structure¶

handler¶
Object Properties:
 
  • multi_face_policy (integer) – Process or not several faces in an input event. Available only for extract-type handlers. Allowed values are 1 or 0.
  • type (user_name) – The handler type. Allowed values are extract or search.
  • grouping_policy (grouping_policy) – The events grouping, persons creation from the groups and attach created persons to Luna API lists rules.
  • descriptor_policy (descriptor_policy) – The attach extracted descriptor to Luna API lists rules.
  • search_policy (search_policy) – The similar descriptors search rules. Required for search-type handlers.
  • handler_id (uuid4) – The handler id.
  • extract_policy (extract_policy) – The descriptor extract rules.
  • person_policy (person_policy) – The person creation from extracted descriptor and attach the descriptor to Luna API lists rules.
  • name (user_name) – The handler name.
JSON Example
{
    "multi_face_policy": 2700,
    "type": "tmendez",
    "grouping_policy": "{grouping_policy object}",
    "descriptor_policy": "{descriptor_policy object}",
    "search_policy": "{search_policy object}",
    "handler_id": "cce506e0-934c-43ba-bc14-46b306b18d24",
    "extract_policy": "{extract_policy object}",
    "person_policy": "{person_policy object}",
    "name": "williamsanna"
}

/events objects’ structure¶

event¶
Object Properties:
 
  • id (uuid4) – The event id. The event id equals the extracted descriptor id if event was created successfully, otherwise the event id is generated randomly.
  • handler_id (uuid4) – The handler id that received the event.
  • persons_lists (_list_uuid4) – A Luna API persons lists list a person generated from the descriptor was attached.
  • error (error_response) – An error if occurred.
  • search_by_group (group_search_result) – The result of search extracted descriptor by recent events in group.
  • group_id (uuid4) – A event group id it belongs.
  • external_id (user_name) – A event external id from request.
  • search (list of search_result) – A search result according to the handler search policy.
  • source (user_name) – A event source from request.
  • extract (face) – The descriptor extract result.
  • tags (_list_string) – Event tags list from request.
  • person_id (uuid4) – A person id the extracted descriptor was attached to.
  • descriptors_lists (_list_uuid4) – A Luna API descriptor lists list the extracted descriptor was attached.
  • descriptor_id (uuid4) – The extracted descriptor id.
  • create_time (iso8601) – The event create time.
  • user_data (user_name) – A user_data the person generated from the descriptor.

/groups objects’ structure¶

group¶
Object Properties:
 
  • descriptors (_list_uuid4) – The list of events (=descriptors) in the group.
  • error (error_response) – A error if occurred.
  • source (user_name) – The events’ source in group.
  • id (uuid4) – The group id.
  • attributes (group_attributes) – The aggregated events’ attributes.
  • external_tracks_id (_list_string) – The events’ external track id list.
  • tags (_list_string) – The events’ tags list.
  • person_id (uuid4) – The group person id if created when group is closed.
  • last_update (iso8601) – The group last update time. It equals group close time if group is closed.
  • handler_id (uuid4) – The group handler id.
  • ttl (integer) – A group time to live common for all groups in the handler. Group closes if no events added during this period. Period in seconds.
  • create_time (iso8601) – The group create time.
  • search (list of search_result_for_group) – The result of search extracted descriptor by recent events in group. The search rules are set in handler.
  • persons_lists (_list_uuid4) – A Luna API persons lists list a person generated from the group was attached.

/tasks_counter object structure¶

tasks_counter¶
Object Properties:
 
  • count (integer) – The current number (=id) of the last created task.
JSON Example
{
    "count": 5237
}

/tasks objects’ structure¶

task¶
Object Properties:
 
  • progress (float) – The task completion percentage.
  • Location (uri) – The done task location.
  • id (integer) – The task id.
  • result (null) – An empty object while task is not done.
  • last_update (iso8601) – The last update time, the task finish time also if the task is finished.
  • create_time (iso8601) – The task create time.
  • task (clusterization_param | hit_top_n_params | cross_matching_params | linker_params | report_params) – The received task, see ‘task_properties’ structures.
  • type (user_name) – The task type.
  • status (user_name) – The task status. Allowed values are started, progress.
JSON Example
{
    "progress": 34096008277191.6,
    "Location": "http://salas-miles.com/explore/blog/category.jsp",
    "id": 6415,
    "result": null,
    "last_update": "1992-01-07T16:42:44",
    "create_time": "2000-05-19T04:22:39",
    "task": {
        "filters": "{task_filters object}",
        "description": "carrie91",
        "objects": "andersontrevor"
    },
    "type": "fshelton",
    "status": "andersonemily"
}

/tasks_done objects’ structure¶

ended_task¶
Object Properties:
 
  • result (clusterizationResult | hitTopResult | crossMatchingResult | linkingResult | reportResult) – The task completion result.
  • progress (float) – The task completion percentage.
  • task (clusterization_param | hit_top_n_params | cross_matching_params | linker_params | report_params) – The received task, see ‘task_properties’ structures
  • type (user_name) – The task type. Allowed values are: clusterization, hit_top_n, reports, cross_matching, linking.
  • id (integer) – The task id.
  • last_update (iso8601) – The last update time, the task finish time also if the task is finished.
  • create_time (iso8601) – The task create time.
  • status (user_name) – The task status. Allowed values are cancelled, failed or done
JSON Example
{
    "result": {
        "success": "{success object}",
        "errors": "{taskerrors object}"
    },
    "progress": 32485610865.7,
    "task": {
        "filters": "{task_filters object}",
        "description": "noblejennifer",
        "objects": "michaelbowman"
    },
    "type": "sarah96",
    "id": 8674,
    "last_update": "1995-01-26T02:38:14",
    "create_time": "2018-07-27T15:07:45",
    "status": "choward"
}

‘task_properties’ structures¶

clusterization_param¶
Object Properties:
 
  • filters (task_filters) – Set of filters to define objects for clusterization.
  • description (user_name) – The user description in the task.
  • objects (user_name) – The objects type to clusterize. One of events, groups or luna_list.
JSON Example
{
    "filters": "{task_filters object}",
    "description": "barnesjoseph",
    "objects": "hernandezdiane"
}
hit_top_n_params¶
Object Properties:
 
  • list_id (uuid4) – The Luna API persons list id.
  • description (user_name) – The user description in the task.
  • top_n (integer) – The n number of positions in the result. Allowed values are 1 to 5.
JSON Example
{
    "list_id": "b5b86d8c-9ec1-4f09-989c-e040b94700b7",
    "description": "vwilson",
    "top_n": 6002
}
cross_matching_params¶
Object Properties:
 
  • references (cross_matching_reference) – The match references objects.
  • threshold (float) – The matching results lower similarity threshold.
  • description (user_name) – The user description in the task.
  • candidates (cross_matching_candidate) – The match candidates objects.
  • limit (float) – The match limit.
JSON Example
{
    "references": "{cross_matching_reference object}",
    "threshold": 3856654402354.1,
    "description": "daniel07",
    "candidates": "{cross_matching_candidate object}",
    "limit": 592599128.0
}
linker_params¶
Object Properties:
 
  • filters (task_filters) – The filters for selecting events and groups for attaching to the Luna API list.
  • list_type (user_name) – The type of the list to which objects will be attached. For groups, only the type ‘persons’ should be set.
  • description (user_name) – The user description in the task.
  • object (user_name) – The objects type to attach. One of events, groups or luna_list.
  • list_data (user_name) – The list_data field for the list that will be created if no list_id provided.
  • list_id (uuid4) – The Luna API list id to attach objects to. If the parameter is not set, the list will be created automatically.
JSON Example
{
    "filters": "{task_filters object}",
    "list_type": "vclark",
    "description": "lisavelazquez",
    "object": "johndoyle",
    "list_data": "edwardjones",
    "list_id": "484b40dd-000b-425d-83a4-9e023a853777"
}
report_params¶
Object Properties:
 
  • description (user_name) – The user description in the task.
  • task_id (integer) – The task id.
  • format (user_name) – The report format. One of csv or pdf.
  • parameters (parameters) – The reporter parameters: the images’ color bounds in pdf-report and save-portraits flag in csv-report.
JSON Example
{
    "description": "marissaduncan",
    "task_id": 584,
    "format": "kevin76",
    "parameters": "{parameters object}"
}

‘task_result’ structures¶

clusterizationResult¶
Object Properties:
 
  • success (success) – The result clusters.
  • errors (taskErrors) – Errors if occurred.
JSON Example
{
    "success": "{success object}",
    "errors": "{taskerrors object}"
}
hitTopResult¶
Object Properties:
 
  • success (hitTopSuccess) – The result hit top n result.
  • errors (taskErrors) – Errors if occurred.
JSON Example
{
    "success": "{hittopsuccess object}",
    "errors": "{taskerrors object}"
}
crossMatchingResult¶
Object Properties:
 
  • matches (list of crossmatchreference) – The result matches list.
JSON Example
{
    "matches": [
        {
            "rossmatchreferenc": "{rossmatchreferenc object}"
        },
        {
            "rossmatchreferenc": "{rossmatchreferenc object}"
        },
        {
            "rossmatchreferenc": "{rossmatchreferenc object}"
        }
    ]
}
linkingResult¶
Object Properties:
 
  • success (abstractSuccess) – Success and failed numbers of objects.
  • errors (taskErrors) – Errors if occurred.
JSON Example
{
    "success": "{abstractsuccess object}",
    "errors": "{taskerrors object}"
}
reportResult¶
Object Properties:
 
  • success (reportSuccess) – Location of completed report.
  • errors (taskErrors) – Errors if occurred.
JSON Example
{
    "success": "{reportsuccess object}",
    "errors": "{taskerrors object}"
}
