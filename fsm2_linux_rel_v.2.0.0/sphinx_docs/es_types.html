Other database objects structures¶

crossMatchReference¶
Object Properties:
 
  • reference (uuid4) – object id
  • source (user_name) – the source of the object, if the reference was an event or group
  • last_update (iso8601) – The last descriptor update time, if the reference was a descriptor
  • tags (list of string) – object tags, if the reference was an event or group
  • user_data (user_name) – person user_data, if the reference was a person
  • candidates (list of crossmatchcandidate) – The match candidates list.
  • create_time (iso8601) – the creation time of the reference, if the reference was a person, event or group
JSON Example
{
    "reference": "c3c070ff-00cd-4399-bc30-d835db6cde84",
    "source": "grodriguez",
    "last_update": "1984-05-02T11:47:02",
    "tags": [
        {
            "trin": "{trin object}"
        },
        {
            "trin": "{trin object}"
        },
        {
            "trin": "{trin object}"
        }
    ],
    "user_data": "shannon75",
    "candidates": [
        {
            "rossmatchcandidat": "{rossmatchcandidat object}"
        },
        {
            "rossmatchcandidat": "{rossmatchcandidat object}"
        },
        {
            "rossmatchcandidat": "{rossmatchcandidat object}"
        }
    ],
    "create_time": "1989-08-09T03:03:24"
}
crossMatchCandidate¶
Object Properties:
 
  • person_id (uuid4) – id of the candidate-person, indicated if the a list of persons was matched
  • similarity (float) – Similarity with the reference descriptor. * 1 means that descriptors are obtained from identical images * numbers close to 1 mean the descriptors a from the same person * numbers close to 0 mean the descriptors from different people
  • id (uuid4) – id of the descriptor, specified if a list of descriptors was matched
  • descriptor_id (uuid4) – descriptor of the candidate person, specified if a list of persons was matched
  • user_data (user_name) – user_data of the candidate-person, specified if a list of persons was matched
JSON Example
{
    "person_id": "0cb1b42f-700f-44e6-90a9-8c3905aa5ee2",
    "similarity": 573646234.45334,
    "id": "d51a3f7b-7c56-4741-b93e-fe1453c89752",
    "descriptor_id": "9c3bd1b3-5a7a-47d2-953a-da0bef7dba7d",
    "user_data": "jessica95"
}

attributes¶
Object Properties:
 
  • age (float) – Age estimation (in years).
  • gender (float) – Gender estimation. * 0 - female * 1 - male
  • eyeGlasses (float) – Estimates are the persons wears eye glasses. * 0 - no glasses * 1 - eye glasses This attribute does not classify eyewear, ie both sunglasses and transparent glasses are reported as 1.
JSON Example
{
    "age": 77411.1586,
    "gender": 18441.9723658,
    "eyeGlasses": 91984354.82722
}

Object Properties:
 
  • age__lt (integer) – upper age limit.
  • create_time__lt (iso8601) – end of the interesting period in which the event occurred
  • age__gt (integer) – the lower age limit.
  • tags (list of string) – a set of tags for filtering.
  • similarity__gt (float) – lower threshold of similary.
  • handler_ids (uuid4) – list of id handlers, as a result, events will be displayed with any of the indicated handlers.
  • create_time__gt (iso8601) – start of the interesting period in which the event occurred

  • sources (list of string) – a list of event sources, as a result, events from any of the specified sources will be output.

search_policy¶
Object Properties:
 
  • search_lists (list of search_list) – list of Luna API lists, which are supposed to match
  • search_priority (integer) – The search by lists priority. * 1 - Search is on all lists, the results are combined. * 2 - the search is in order, if the result of the match on some list has exceeded threshold, the search stops.
JSON Example
{
    "search_lists": [
        {
            "earch_lis": "{earch_lis object}"
        },
        {
            "earch_lis": "{earch_lis object}"
        },
        {
            "earch_lis": "{earch_lis object}"
        }
    ],
    "search_priority": 5948
}

Object Properties:

search_list¶
Object Properties:
 
  • list_id (uuid4) – the Luna API list id
  • list_type (user_name) – in the Luna API list type, sheet of descriptors or persons
  • threshold (float) – the threshold for filtering search results and recognizing a successful search

grouping_policy¶
Object Properties:
 
  • age (integer) – Age aggregator for the group of events. Now the only kind of aggregator is available - mean value.
  • grouper (integer) – The events grouping policy: * 1 - by the face track id from which the image was obtained; * 2 - by the similarity to last events from this handler and the given source; 3 - mixed grouping, by id of the track and similarity.
  • create_person_policy (create_person_policy) – The creating a person from the group policy. The person is created when the group closes. It is impossible to create a person from the group, if a person is created from the descriptor.
  • ttl (integer) – A group time to live. Group closes if no events added during this period. Period in seconds.
  • threshold (float) – The events grouping similarity threshold for the second and third grouping policy. The new event is assigned to the group if the new event is similar to any of the events from the group with similarity more than this number. If there are several proper groups, then event is attached to the one to which the most similar descriptor belongs.
  • gender (integer) – An events’ group gender aggregator. Now two types of aggregators are available: * 1 - maximum deviation from 0.5; * 2 - average value.
  • search (integer) – The search results aggregator. Now there are two types of aggregators available: * 1 - top 3 results by similarity; * 2 - top 3 results by the majority voting.
